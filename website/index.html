Why it exists

MCP is powerful. Big MCP servers are heavy.

Native MCP can expose long tool descriptions and schemas on every model call. That is fine for small tools, but expensive for SaaS, document, and workflow MCP servers.

ToolCapsule keeps MCP as the capability layer and generates a compact Agent Skill as the workflow layer. Full schemas are loaded only when needed.

01

Discover

List tools as compact summaries instead of injecting every schema into every turn.

02

Capsule

Put large arguments, Markdown, and JSON payloads in local files.

03

Call

Execute MCP tools through a reproducible CLI and save request/response artifacts.

04

Patch & retry

When calls fail, patch the file instead of asking the model to regenerate everything.

Before

Regenerate the whole tool call

  • Full MCP schema stays close to the prompt.
  • Large payloads are recreated by the model.
  • Retries may accidentally change correct content.

After

Patch local artifacts and replay

  • Schema is lazy-loaded only when needed.
  • Payloads live in args.json and content files.
  • Retries are deterministic and auditable.
